## 1. Vision & Mission

**Vision**: Become the go-to platform where wishlists turn into reality - whether you're saving for yourself, sharing with loved ones, or celebrating life's moments together.

**Mission**: Bridge the gap between wanting and getting by providing one beautiful place to save products from anywhere, track prices, coordinate gifts, and celebrate every occasion.

**Tagline Options**:
- "From Wish to Cart" (primary)
- "Every Wish, One Place"
- "Save. Share. Celebrate."

**Core Value Proposition**: WishlistCart is the only platform that combines universal product saving from any store, intelligent price tracking, and social gift coordination for every life occasion - all in one place.

---

## 2. Product Overview

WishlistCart.com is a hybrid platform with two complementary pillars:

### Pillar 1: Universal Wishlist (Personal Shopping)
Save products from ANY online store into one organized hub. Track prices, get deal alerts, and make smarter purchase decisions.

### Pillar 2: Gift Registry & Social Gifting (Occasions)
Create and share wishlists for any life event. Coordinate gifts with family and friends to avoid duplicates, pool money for expensive items, and celebrate together.

**Why Both Together**: The wishlist drives daily engagement (users saving items regularly), while the registry drives viral growth (users sharing lists and inviting others). Personal wishlists convert to registries when occasions arise. Each pillar feeds the other.

#### B. Universal Wishlist (Core)
- [ ] **Add Item via URL**: Paste any product URL, auto-extract title, image, price, store name
- [ ] **Manual Item Add**: Add custom items (name, description, price, image upload, link)
- [ ] **Multiple Wishlists**: Create unlimited named wishlists (e.g., "Tech Gadgets", "Kitchen", "Books")
- [ ] **Default Wishlist**: Quick-add items without choosing a list
- [ ] **Item Details**: Product name, image(s), current price, store, date added, notes, priority level
- [ ] **Priority Levels**: Must-have, Nice-to-have, Dream item (or 1-5 stars)
- [ ] **Drag & Drop Reorder**: Rearrange items within lists
- [ ] **Move/Copy Between Lists**: Transfer items across wishlists
- [ ] **Item Notes**: Personal notes on each item (size, color preference, "the blue one not the red")
- [ ] **List Views**: Grid view (visual) and List view (compact)
- [ ] **Sort & Filter**: By price, date added, priority, store, category
- [ ] **Search**: Search across all wishlists
- [ ] **Archive**: Archive old wishlists without deleting
- [ ] **Tags/Categories**: Tag items for cross-list organization

#### C. Product Data Extraction
- [ ] **URL Scraper**: Extract product data from pasted URLs (title, price, image, description)
- [ ] **Supported Stores** (initial): Amazon, Walmart, Target, Best Buy, Etsy, eBay, Nordstrom, IKEA, Apple, Nike, Adidas, Sephora, Ulta, Home Depot, Wayfair, Costco, Macy's, and generic OpenGraph fallback
- [ ] **Image Extraction**: Pull primary product image, allow user to choose from multiple
- [ ] **Price Extraction**: Get current price, detect sale prices
- [ ] **Fallback**: If scraping fails, allow manual entry with URL stored
- [ ] **Product Update**: Periodically re-check URLs for price/availability changes

#### D. Sharing & Privacy
- [ ] **Share Link**: Generate shareable link for any wishlist
- [ ] **Privacy Levels**: Private (only me), Shared (link only), Public (discoverable)
- [ ] **Share via**: Copy link, Email, WhatsApp, Facebook, Twitter/X, iMessage, SMS
- [ ] **QR Code**: Generate QR code for each wishlist (great for events)
- [ ] **Embed Widget**: Embeddable wishlist widget for blogs/websites
- [ ] **View Permissions**: View-only vs. can-add (collaborative lists)
- [ ] **Password Protection**: Optional password on shared lists

#### E. Gift Coordination (for shared lists)
- [ ] **"I'm Buying This" Button**: Gift-givers mark items as claimed (hidden from list owner)
- [ ] **Purchase Confirmation**: Mark items as actually purchased
- [ ] **Surprise Mode**: All purchase info hidden from the wishlist owner
- [ ] **Avoid Duplicates**: Visual indicator when item is claimed (visible to other gift-givers only)
- [ ] **Gift Message**: Attach a message when claiming/buying an item
- [ ] **Anonymous Gifting**: Option to hide who bought what

## 7. Technical Architecture

### 7.1 Tech Stack

| Layer | Technology | Rationale |
|-------|-----------|-----------|
| **Framework** | Next.js 15 (App Router) | SEO, SSR, API routes, Vercel-native |
| **Language** | TypeScript | Type safety, better DX, fewer bugs |
| **Styling** | Tailwind CSS + shadcn/ui | Rapid UI development, consistent design |
| **Database** | PostgreSQL (Supabase or Neon) | Relational data, managed hosting, free tier |

| **Auth** | NextAuth.js v5 (Auth.js) | Google/Apple OAuth, email/password, sessions |
| **File Storage** | Cloudflare R2 or Supabase Storage | Product images, user avatars, cheap/free |
| **Payments** | Stripe | Group gifting pools, subscriptions |
| **Email** | Resend | Transactional emails, price alerts, digests |
| **Search** | Meilisearch or Algolia | Fast product/wishlist search |
| **Caching** | Redis (Upstash) | Session cache, rate limiting, price cache |
| **Queue/Jobs** | Inngest or Trigger.dev | Price checking cron, email digests, scraping |
| **Hosting** | Vercel | Zero-config deployment, edge network, scaling |
| **Analytics** | PostHog or Plausible | Privacy-friendly user analytics |
| **Monitoring** | Sentry | Error tracking, performance monitoring |
| **CDN** | Vercel Edge Network | Global content delivery |

## 8. Monetization Strategy

### Revenue Streams (Priority Order)

#### Stream 1: Affiliate Commissions (Day 1)
The primary and immediate revenue source. Every outbound product link uses affiliate tracking.

| Network | Commission Rate | Coverage |
|---------|----------------|----------|
| Amazon Associates | 1-10% by category | Amazon products |
| ShareASale | 5-30% varies | 16,000+ merchants |
| CJ Affiliate | 3-20% varies | Major retailers |
| Rakuten | 3-15% varies | Premium brands |
| Impact | 5-20% varies | DTC brands |
| Individual programs | 5-20% | Target, Walmart, etc. |

**Implementation**: When a user clicks "Buy" on any item, redirect through our affiliate link. Track clicks and conversions. Estimated revenue: $0.50-$5.00 per conversion.

**Projected**: $0.02-0.05 per active user per month initially, growing to $0.50-2.00 as engagement increases.

#### Stream 2: Premium Subscription (Month 3+)

**WishlistCart Pro** - $4.99/month or $39.99/year

| Feature | Free | Pro |

## 11. Growth & Marketing Strategy

### 11.1 Acquisition Channels

| Channel | Strategy | Cost | Timeline |
|---------|----------|------|----------|
| **SEO/Content** | Gift guides, registry checklists, blog | Free (time) | Month 2+ (slow build) |
| **Viral/Sharing** | Users share wishlists with friends/family | Free | Day 1 |
| **Social Media** | Pinterest, Instagram, TikTok gift content | Free-Low | Month 1+ |
| **Product Hunt** | Launch on PH for initial spike | Free | Month 2-3 |
| **Reddit/Forums** | Helpful posts in r/gifts, r/weddings, etc. | Free | Month 1+ |
| **Browser Extension** | Chrome Web Store listing, organic installs | Free | Month 3+ |
| **Influencer** | Gift/lifestyle micro-influencers | $50-500/post | Month 4+ |
| **Paid (later)** | Google Ads on gift-related keywords | $0.50-3/click | Month 6+ |

### 11.2 Viral Loops

1. **Wishlist Sharing**: User creates list → shares with 5-10 friends/family → friends visit → some sign up → create their own lists. **Viral coefficient target: 0.3-0.5**

2. **Registry Invites**: Couple creates wedding registry → shares with 100-200 guests → guests see the platform → some create accounts for their own occasions. **Viral coefficient target: 0.1-0.2 per guest**

3. **Group Gifting**: Organizer starts pool → invites 5-15 contributors → contributors see platform → some sign up. **Viral coefficient target: 0.2-0.3**

4. **Gift Guide Sharing**: User finds great gift guide → shares on social media → drives traffic. Organic SEO amplifies this.

### 11.3 Retention Strategy

- **Weekly digest emails**: Price changes, deals on wishlisted items
- **Occasion reminders**: "Mom's birthday is in 2 weeks!"
- **Price drops**: Instant notification when items get cheaper
- **Seasonal campaigns**: Pre-holiday email series
- **Feature education**: Drip campaign teaching features post-signup
- **Re-engagement**: "You haven't checked your wishlist in 30 days" emails

## 13. Risk Mitigation

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| **Amazon/major retailers block scraping** | Medium | High | Use official Product Advertising APIs, OpenGraph fallback, manual entry fallback |
| **Affiliate programs change terms** | Medium | High | Diversify across 10+ networks, build subscription revenue as backup |
| **Established player copies features** | Medium | Medium | Move fast, build community moat, focus on UX differentiation |
| **Low initial traffic** | High | Medium | Heavy SEO investment, viral features from day 1, content marketing |
| **AI chatbots replace wishlist apps** | Low-Medium | Medium | Lean into social/human elements AI can't replicate; use AI as feature enhancer |
| **User data breach** | Low | Very High | Security best practices, encryption, regular audits, minimal data collection |
| **Stripe/payment issues** | Low | High | Keep payment scope small initially, comply with all regulations |
| **Extension rejected from stores** | Low | Medium | Follow all store policies strictly, have web-only fallback |
| **Seasonal revenue dependency** | Medium | Medium | Build year-round use cases (personal wishlists, price tracking) |
| **User acquisition cost too high** | Medium | Medium | Rely on organic/viral channels, avoid paid ads until profitable |
